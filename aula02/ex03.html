Por que a primeira conta mostrou um resultado errado?

Isso aconteceu porque ao pedir a resposta de um prompt como o valor da variável, ele é considerado como uma string, e o sinal de "+" em JavaScript também é usado para concatenação além da adição, sendo assim, '1' + '1' (ambos considerados strings) resultam em '11' (uma única string formada das duas strings originais)

Para resolver esse "problema" precisamos converter a variável de uma string para um number, a forma mais fácil de fazer isso é adicional Number() em nossa string, tudo que estiver dentro de Number() será considerado um número (real ou inteiro)

Em algumas situações, pode ser útil que o números seja especificamente inteiro ou real, nesses casos, ao invés de usarmos Number() iremos usar Number.parseInt(para números inteiros) e Number.parseFloat(para números reais)


Confira agora o código usado para efetuar essas contas:

var n1 = window.prompt('Digite um número:') 
var n2 = window.prompt('Digite outro número: ')
c = n1 + n2
alert('O resultado da soma é: ' + c + ' Bom... isso parece errado não é mesmo? Entenda o porquê a seguir...')
var n3 = Number(window.prompt('Digite um número: '))
var n4 = Number(window.prompt('Digite outro número: '))
s = n3 + n4
alert('O resultado da soma é: ' + s + ' Agora sim! Já vou explicar o que aconteceu na primeira conta, só preciso de mais alguns números...')
var n5 = Number.parseInt(window.prompt('Digite um valor Inteiro:'))
var n6 = Number.parseFloat(window.prompt('Agora um valor real: '))
s2 = n5 + n6
alert('O resultado da soma é: ' + s2 + ' Pronto, consegui entender o problema, vou te explicar agora...')
